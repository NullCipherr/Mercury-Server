Servidor HTTP/1.1 com controle fino de memória e métricas em tempo real

O Mercury Server é uma base de backend low-level para quem quer performance previsível, parser manual e arquitetura clara para evoluir em produção sem acoplar frameworks.

Feito para engenharia de sistema, não para esconder complexidade

Arquitetura orientada a previsibilidade operacional: menos mágica, mais controle de I/O, limites explícitos e fluxo de request transparente para debug e evolução.

Parser HTTP manual

Sem framework web. Parsing direto com limites de header/body configuráveis e foco em robustez.

Pool thread-safe

Fila concorrente desacopla accept do processamento e reduz churn de threads.

Métricas embutidas

Endpoint /metrics com contadores de requests, erros e latência média.

Hardening básico

Timeouts por conexão, validação de payload e proteção contra path traversal no static server.

Operação prática

Execução via Zig, Makefile e Docker Compose para ambiente local ou pipeline CI/CD.

Base para crescer

Código modular em Zig para evoluir keep-alive, streaming de arquivos e observabilidade avançada.

Contribuição: entre rápido e com contexto

Quer melhorar parser, observabilidade, testes ou docs? O projeto está aberto para PRs técnicas e incrementais. O fluxo é simples e já documentado para facilitar onboarding.

  1. 1. Leia o guia em CONTRIBUTING.md e alinhe escopo da mudança.
  2. 2. Faça um fork e crie branch com prefixo descritivo (ex.: feat/...).
  3. 3. Rode os checks locais com zig build test e scripts de validação.
  4. 4. Abra PR com contexto técnico, trade-offs e evidências de teste.
